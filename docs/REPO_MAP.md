# Treaty-Lab — Repository Map

> Companion to `ARCHITECTURE.md`, `SYSTEM_FLOW.md`, `DATABASE_SCHEMA.md`.
> Inventory of every directory and significant file with a classification and
> a recommended action. Intended to drive the cleanup pass that precedes a
> production push.

## Legend

**Classification** — what the artifact is:

| Tag | Meaning |
|-----|---------|
| **core** | Production code or content the app depends on at runtime |
| **prototype** | Working code that's intentional but should be hardened before prod |
| **scratch** | Working notes, one-shot scripts, commit-message drafts — not for prod |
| **docs** | Documentation (Markdown), not loaded at runtime |
| **config** | Build/tool configuration |
| **ignore** | Generated, secret, or vendor — typically gitignored |

**Action** — what to do with it:

| Tag | Meaning |
|-----|---------|
| **stay** | Keep as-is in its current location |
| **move** | Relocate to a more appropriate folder (target noted) |
| **rename** | Keep the file but change its name |
| **remove** | Delete from the repo (ensure gitignored if regenerated) |
| **refactor** | Keep in place but materially rewrite |
