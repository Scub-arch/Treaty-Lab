Pilot v0.2.4: cited-sources index, methodology FAQ, project lineage tree

Three depth additions, all repurposed from generic-SaaS component templates
the user pointed at but rebuilt for Treaty-Lab content (skipping the
verbatim Ruixen / Mountain Adventure / Industry Recognition flavor).

1. /sources cited-evidence index (new route)
   - Dense row-style index of all 36 evidence items, grouped by primary
     tag (Law / Finance / Water / Power / Government / Community /
     Company; Market omitted because empty).
   - Each row: source-type code + title/author + publishedAt +
     evidence-strength badge + link to /evidence/[slug].
   - Added to sidebar Research group as "Cited Sources" (SRC).
   - Pattern-borrowed from the supplied list-2 template; content,
     icons, palette, and copy entirely rewritten.

2. Methodology FAQ on /explainers (new section)
   - 10 real Treaty-Lab questions covering the claim-kind taxonomy
     (FACT / RISK / QUESTION / ASSUMPTION / NEEDS VALIDATION),
     evidence-confidence rating, Yahey cumulative-effects doctrine,
     FPIC origins (UNDRIP Articles 19/32, UN Declaration Act, BC DRIPA),
     AB Water Act silence on Section 35, why Cedar LNG's page is more
     detailed than Site C/TMX, how the pilot projects were selected,
     how to read the Cross-Domain Severity Composite, whether any
     data is real-time, and Treaty Archive vs Evidence Library scope.
   - Uses the existing Base UI accordion primitive (already in the
     project since v0.1 via the shadcn v4 base-ui migration).
   - Notable API note: Base UI's Accordion does NOT take a `type` prop
     (multiple-open is the default), unlike Radix's. Fixed during
     integration.

3. /projects/[slug] lineage tree (new full-width panel)
   - Server component using d3-hierarchy for tree layout + plain SVG
     for rendering (no visx, no client JS for the layout).
   - 3-level structure: Project (root, --brand amber) -> 2 group
     anchors (Parties, Primary Sources) -> leaves.
   - Party leaves color-coded by role: amber for contesting,
     emerald for consenting, neutral for regulators / financiers.
   - Source leaves link directly to /evidence/[slug].
   - Height computed from leaf count; minimum 280px; ~28px per leaf.
   - For Cedar LNG: 16 nodes (1 root + 2 group + 8 parties + 5 sources),
     15 connecting paths, 32 text labels.

Explicitly NOT integrated from the source dumps:
- ruixen-accordian02 (FAQ with refunds/plan-upgrade questions — generic)
- story.tsx (Instagram-style story viewer — zero domain fit)
- stories-carousel.tsx (Embla carousel of Unsplash travel photos —
  spec explicitly forbids stock-photo sections)
- dendrogram.tsx via visx (chose d3-hierarchy + plain SVG instead to
  keep recharts as the only chart library; visx adds 4 separate
  packages with overlapping functionality)
- list-2 verbatim ("Industry Recognition / Innovation Prize" awards —
  pattern repurposed for cited-sources index instead)

Deps: d3-hierarchy + @types/d3-hierarchy. Separator primitive was
already in the project from v0.1.
