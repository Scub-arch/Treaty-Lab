Pilot v0.2.1: Geographic Intelligence globe on Command Center

Add a cobe-canvas globe to the Command Center as a Geographic Intelligence
panel, paired with the cross-domain severity radar in a 2-column layout.

The globe is an institutional rework of the
`ui-components-reference/cobe-globe-weather` reference:
- Strips the demo's weather-emoji sticker styling (filters, experimental
  CSS Anchor Positioning, floating animations) — wrong tone for this
  domain.
- Replaces weather markers with real Treaty-Lab anchors:
  - Cedar LNG (Kitimat — Haisla territory)
  - Coastal GasLink origin (Dawson Creek)
  - Site C (Peace River — Treaty 8)
  - Blueberry River First Nations territory (Yahey claim area)
  - TMX origin (Edmonton) and terminus (Burnaby — Tsleil-Waututh)
  - Capital Power Genesee data-centre proposal (Treaty 6)
  - AESO data-centre cluster (Calgary, 3,533 MW)
- Adds project-corridor arcs: TMX (Edmonton -> Burnaby), CGL
  (Dawson Creek -> Kitimat feeding Cedar LNG).
- Subtle warm-amber markers matching the --brand token.
- Includes a paired marker list with mono-spaced lat/lon coordinates
  beneath the globe for accessible / non-visual readers.

Explicitly NOT integrated from the same reference directory:
- `molten-core-shader` (three.js fragment-shader red/orange lava
  background — wrong tone for an institutional Indigenous-led
  intelligence terminal; heavy bundle weight; decorative-only).
- `aero-hero-2` (stock-photo unsplash hero with lime-green CTA and
  "Sustainable Solutions for a Better Future" tagline — directly
  violates the spec's "avoid stock-photo / activist / sales-landing
  visuals"; also depends on @base-ui/react which conflicts with the
  shadcn Button shape used everywhere else).

Bug fix during integration: the reference's fade-in mechanism set
`opacity: 0` via inline JSX style and then tried to set `opacity: 1`
imperatively inside a requestAnimationFrame callback. React re-applies
inline styles on every render, so the canvas stayed invisible after
HMR / re-renders. Removed the fade-in entirely.

Other: cobe library uses `from`/`to` for arcs in its current TS types
(not `start`/`end` as in the reference) — translated at the call site.

Deps added: cobe.
