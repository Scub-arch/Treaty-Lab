Pilot v0.2.3: 4-quadrant Command Center overview

Replace the existing 2-column Geographic + Composite section on Command
Center with a denser 4-quadrant overview, inspired by the layout pattern
of a generic SaaS marketing section the user pointed at but rebuilt
end-to-end with Treaty-Lab-appropriate content:

  CMD · GEO         (TL)  Cobe globe with project + territory markers
                          and project-corridor arcs (already shipped in
                          v0.2.1; slot-in)
  CMD · COMPOSITE   (TR)  Cross-domain severity radar + per-domain
                          readings list (shipped in v0.2; slot-in)
  CMD · TREND       (BL)  NEW LoadGrowthTrend area chart — AESO
                          large-load (data-centre) interconnection queue
                          growth over 2025 (Mar 11,834 MW / 22 projects
                          -> Jun 16,229 MW / 29 projects -> YE 20,000+
                          MW), with sidecar references to Alberta winter
                          peak (12,785 MW) and Phase I cap actually
                          cleared (1,200 MW). All cited.
  CMD · PRJ         (BR)  2x2 sub-grid of the 4 pilot project assessment
                          cards (Cedar LNG, CGL, Site C, TMX) with an
                          "OPEN ALL" link to /projects.

Explicitly NOT integrated from the source component:
- The "Ruixen" SaaS brand and copy ("Ready to use blocks", "Pro User
  Feedback", "Figma plugin connected").
- The dotted-map dependency (cobe globe already provides Treaty-Lab's
  geographic visualization).
- The duplicate copy of the shadcn chart primitive that ships in the
  source (would have collided with the existing src/components/ui/chart.tsx
  ported in v0.2.1, and contained a known bug from the original ui-app
  session — template-string newlines inside .join("\n") instead of escaped
  \n).
- Placeholder image paths (/ruixen_dark.png, /1.png, /2.png) — would 404
  and require stock photography the spec explicitly forbids.
- The fade-in scaleUp keyframes — only useful for the source's notification
  cards, which we don't include.

Bug fixed during integration: LoadGrowthTrend originally used
`flex-1 min-h-[140px]` on its chart wrapper. Recharts ResponsiveContainer
measured 429x0 because the parent panel body had no defined height to
distribute. Switched to explicit `h-[180px]` — chart paints reliably.

Recharts Tooltip formatter signature in this version accepts ValueType
that can be undefined, so the typed formatter has to narrow internally
rather than assert `value: number`.

No new npm deps.
