Pilot v0.2.2: silence Dependabot alerts via npm overrides

Resolve the two moderate-severity Dependabot vulnerabilities flagged on
main after the v0.2 push:

  GHSA-92pp-h63x-v22m  @hono/node-server < 1.19.13 — middleware bypass
                        via repeated slashes in serveStatic
                        (transitive through prisma -> @prisma/dev)

  GHSA-qx2v-qp2m-jg93   postcss < 8.5.10 — XSS via unescaped </style>
                        in CSS Stringify Output
                        (transitive through next)

Both packages are already on their latest published versions (prisma
7.8.0; next 16.2.6) — upstream hasn't released a patch that bumps
the pinned transitive deps. Used npm `overrides` in package.json to
force the patched versions:

    "overrides": {
        "@hono/node-server": "^1.19.13",
        "postcss": "^8.5.10"
    }

After re-install:
  @hono/node-server -> 1.19.14 (was 1.19.11)
  postcss           -> 8.5.15  (was 8.4.31)
  npm audit         -> 0 vulnerabilities (was 5 moderate)

Neither vulnerability actually affects Treaty-Lab's surface area
(we don't expose Prisma's Hono-based HTTP server, and we don't
take user input -> CSS), so this is a clean-up patch rather than
a risk-posture change. tsc --noEmit clean; /archive (Prisma-backed)
renders with all 12 treaty records intact.
